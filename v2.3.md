# v2.3 — DataCache Structural Fix (Nautilus-Style Single Writer)

## Context

The engine has a race condition where 4 separate writers mutate `app_state.positions` and 3 writers mutate `app_state.account_state` concurrently. The core bug is a TOCTOU race in `exchange.py:fetch_positions()` (lines 189-235): it reads existing positions under lock, processes outside lock, then writes under a second lock — any WS update between reads and writes is silently overwritten. Additionally, `_apply_position_updates_normalized` appends without lock, platform bridge writes account state without lock, and stale REST data triggers false `trade_closed` events.

The fix: create a `DataCache` class that is the **sole writer** to all mutable state. All data sources (WS, REST, platform) submit updates to DataCache, which resolves conflicts using timestamps and publishes events only after mutation.

## Phase 1: DataCache Core + Open Positions

**Goal**: Fix the TOCTOU race, unlocked appends, and false trade_closed events.

### Create: `core/data_cache.py` (~250 lines)

```python
class UpdateSource(Enum):
    WS_USER = "ws_user"
    WS_MARKET = "ws_market"  
    REST = "rest"
    PLATFORM = "platform"
    INTERNAL = "internal"

@dataclass
class VersionedState:
    sequence: int = 0
    source: UpdateSource = UpdateSource.INTERNAL
    timestamp_ms: int = 0

class DataCache:
    _lock: asyncio.Lock
    _event_bus: EventBus
    _positions: List[PositionInfo]
    _positions_version: VersionedState
```

**Key methods:**
- `apply_position_update_incremental(source, norm_positions, balances, ts_ms)` — WS delta updates. Acquires lock, applies changes atomically, publishes events after
- `apply_position_snapshot(source, positions, ts_ms)` — REST/platform full replace. Conflict resolution: reject REST if WS updated within 5s. Detect genuine closures by comparing against current state under same lock
- `_should_accept(current_version, incoming_source, incoming_ts_ms)` — Reject REST if WS/Platform data is fresher than 5s

**Conflict resolution rule:**
```
if current.source in (WS_USER, PLATFORM) and incoming == REST:
    if (incoming_ts_ms - current.timestamp_ms) < 5000ms:
        return False  # reject stale REST
return True
```

## Phase 2: Event Publishing Migration + History/Logs

- **orders**: Written by `order_manager.process_order_snapshot()` — DB-only, not in app_state

## Backward Compatibility Strategy

All **reads** go through `AppState` properties that delegate to DataCache:
- **15+ files** reading `app_state.positions` — ZERO changes needed
- **13+ files** reading `app_state.account_state` — ZERO changes needed
- **8+ files** reading market data caches — ZERO changes needed
- All API routes, DB mixins, Jinja templates — ZERO changes needed

Only **writers** change, and they change incrementally per phase.

Legacy setter on `app_state.positions = [...]` logs deprecation warning to catch any un-migrated writer during development.
