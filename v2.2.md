# v2.2 — Account Management & Connections Redesign

## Context

The engine currently has a working multi-account system (encrypted SQLite, CCXT factory, account registry) but several limitations:

- **Fees are global** — hardcoded `MAKER_FEE=0.0002`, `TAKER_FEE=0.0005` in `config.py:29-31`, Binance-specific
- **Risk params are global** — single `data/params.json`, shared across all accounts, intentionally NOT reset on account switch (`core/state.py:213-225`)
- **3rd party API keys hardcoded** — FRED, Finnhub, CoinGecko keys live in `.env`/`config.py`, not manageable from UI
- **UI mixes concerns** — `templates/params.html` contains both risk params form AND account management table
- **No easy onboarding** — adding a new exchange account requires knowing fee structures upfront; no template/copy mechanism

**Goal**: Make accounts self-contained (own params, own fees), externalize all API keys into encrypted DB with UI management, and redesign the frontend to support this cleanly.

---

## Phase 0 — Security Hardening (Pre-requisite)

These fixes address existing vulnerabilities that v2.2 would amplify. Must be done first or alongside Phase 1.

### 0A. CCXT Exception Sanitization

CCXT exceptions can embed API keys in request URLs. The engine uses `log.exception()` / `exc_info=True` in 10+ locations, serializing full tracebacks to `data/logs/risk_engine.jsonl`.

**Add utility in `core/crypto.py`:**
```python
import re

def safe_exchange_error(e: Exception) -> str:
    """Strip credential fragments from CCXT error messages."""
    msg = str(e)
    return re.sub(r'(apiKey|secret|signature)=[^&\s]+', r'\1=***', msg)
```

**Affected files — replace `log.exception(...)` with `log.error("...: %s", safe_exchange_error(e))`:**
- `core/ws_manager.py:264`
- `core/handlers.py:72,143,163`
- `core/event_bus.py:94`
- `core/regime_fetcher.py:111,455`

### 0B. Fix Silent Decryption Failure

`core/crypto.py` `decrypt()` returns `""` on `InvalidToken` with no log. If ENV_MASTER_KEY rotates, all credentials silently become empty.

**Fix:**
```python
except (InvalidToken, Exception) as e:
    log.error("Decryption failed (key mismatch or corruption): %s", type(e).__name__)
    return ""
```

### 0C. Credential Audit Log

Log credential lifecycle events (add/update/delete) to `data/logs/audit.jsonl`. Never log plaintext credentials. Log: timestamp, action, account/provider name.

**Add to:** `account_registry.add_account()`, `update_account()`, `delete_account()`, and all new `connections_manager` mutators.

### 0D. Atomic Account Activation

`db_settings.py` `set_active_account()` uses two separate UPDATE statements. Wrap in transaction:

```python
await self._conn.execute("BEGIN")
await self._conn.execute("UPDATE accounts SET is_active=0")
await self._conn.execute("UPDATE accounts SET is_active=1 WHERE id=?", (account_id,))
await self._conn.commit()
```

## Phase 1 — Database Schema Changes

### 1A. `account_params` table

Store per-account risk parameters in DB instead of global JSON file.

```sql
CREATE TABLE IF NOT EXISTS account_params (
    account_id  INTEGER NOT NULL,
    key         TEXT    NOT NULL,
    value       REAL    NOT NULL,
    PRIMARY KEY (account_id, key),
    FOREIGN KEY (account_id) REFERENCES accounts(id) ON DELETE CASCADE
);
```

**Files to modify:**
- `core/database.py` — add CREATE TABLE in `_create_tables()`
- `core/db_settings.py` — add `get_account_params(account_id)`, `set_account_params(account_id, params_dict)`, `set_account_param(account_id, key, value)`

### 1B. Add new columns to `accounts` table

```sql
ALTER TABLE accounts ADD COLUMN maker_fee REAL NOT NULL DEFAULT 0.0002;
ALTER TABLE accounts ADD COLUMN taker_fee REAL NOT NULL DEFAULT 0.0005;
ALTER TABLE accounts ADD COLUMN environment TEXT NOT NULL DEFAULT 'live'
    CHECK (environment IN ('live', 'paper', 'testnet'));
ALTER TABLE accounts ADD COLUMN key_version INTEGER NOT NULL DEFAULT 1;
```

- **`environment`** — prevents "fat finger" accidents. UI shows colored badge: `[LIVE]` red, `[PAPER]` yellow, `[TESTNET]` blue. Testnet accounts use CCXT's `set_sandbox_mode(True)`.
- **`key_version`** — future-proofs ENV_MASTER_KEY rotation. When key rotates, re-encrypt rows incrementally by version.

**Files to modify:**
- `core/database.py` — add ALTER TABLE in idempotent migrations section

### 1C. `connections` table

For 3rd party API keys (FRED, Finnhub, CoinGecko, future providers).

```sql
CREATE TABLE IF NOT EXISTS connections (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    provider    TEXT    NOT NULL UNIQUE,   -- 'fred', 'finnhub', 'coingecko', etc.
    label       TEXT    NOT NULL,          -- 'Federal Reserve (FRED)'
    api_key_enc TEXT    NOT NULL DEFAULT '',
    extra_enc   TEXT    NOT NULL DEFAULT '', -- JSON blob for additional fields (encrypted)
    is_active   INTEGER NOT NULL DEFAULT 1,
    key_version INTEGER NOT NULL DEFAULT 1, -- for future master key rotation
    created_at  TEXT    NOT NULL DEFAULT (datetime('now'))
);
```

**Files to modify:**
- `core/database.py` — add CREATE TABLE

### 3F. Auto-Fetch Fees on Connection Test

When `test_connection()` succeeds, also fetch the account's actual fee tier from the exchange.

**All exchange calls must have a 30-second timeout:**
```python
try:
    result = await asyncio.wait_for(
        loop.run_in_executor(pool, ex.fetch_trading_fees),
        timeout=30.0
    )
except asyncio.TimeoutError:
    log.warning("Fee fetch timed out for account %d, using defaults", account_id)
    # Keep existing fee values
```

Apply the same `asyncio.wait_for(timeout=30)` pattern to:
- `test_connection()` → `fetch_time()` (currently has NO timeout — can hang indefinitely)
- `fetch_exchange_info()` during startup (same hang risk)
- The new `fetch_trading_fees()` call

**Cache fee results** — `fetch_trading_fees()` is heavy (Binance returns all pairs). Only fetch on explicit "Test Connection" click, not on every account load.

Not all exchanges support `fetch_trading_fees()` — fallback to config defaults if unavailable.

**Files to modify:**
- `core/account_registry.py` — `test_connection()` enhancement + timeout
- `core/exchange_factory.py` — optional helper
- `core/exchange.py` — add timeout to `fetch_exchange_info()`

## Phase 4 — Backend: Connections Manager

### 4A. `core/connections.py` — New Module

Singleton `connections_manager` similar to `account_registry`:

```python
class ConnectionsManager:
    _cache: Dict[str, dict]  # provider → {label, api_key, extra, is_active}

    async def load_all()
    async def get(provider: str) → Optional[str]  # returns decrypted api_key
    async def upsert(provider, label, api_key, extra=None)  # audit-logged
    async def delete(provider)                               # audit-logged
    async def test(provider) → dict  # 10s timeout, returns {"status": "ok"|"error", ...}
    def list_connections() → List[dict]  # returns api_key_hint (last 4 chars), NEVER full key
```

**Resilience requirements for `test()`:**
- 10-second timeout per provider (FRED/Finnhub/CoinGecko are external HTTP calls)
- Graceful failure — return `{"status": "error", "msg": "..."}`, never crash
- No retry — it's user-triggered, let them click again
- Proxy: do NOT apply `HTTP_PROXY` to data provider calls (proxy is exchange-specific)

**Startup:** `connections_manager.load_all()` called in `main.py` lifespan after `account_registry.load_all()`.

### 4B. Config Fallback Chain

Change config.py to check connections DB first, then fall back to .env:

```python
# config.py
def get_api_key(provider: str) -> str:
    """DB first, then .env fallback."""
    from core.connections import connections_manager
    key = connections_manager.get_sync(provider)
    if key:
        return key
    return os.getenv(f"{provider.upper()}_API_KEY", "")
```

**Files to modify:**
- `config.py` — FRED_API_KEY, FINNHUB_API_KEY, COINGECKO_API_KEY become properties/functions
- All consumers already read from `config.*` so they auto-pick up the change

## Phase 8 — Account Switch Flow Update

**Current:** Stop WS → Reset state → Switch exchange → `asyncio.create_task(_reinit())` → return immediately

**Problems with current flow:**
- `_reinit()` runs as fire-and-forget task — if it fails, engine is bricked (`is_initializing = True` forever)
- No rollback to previous account on failure
- Client gets "ok" response before reinit actually succeeds

**New flow — `await` reinit with rollback:**

```python
async def activate_account(account_id):
    old_id = app_state.active_account_id
    # 1. Stop WS → Reset state → Switch exchange
    # 2. Load per-account params from account_params table
    # 3. Update app_state.params + fees
    try:
        await _reinit()  # AWAIT, not create_task
    except Exception as e:
        log.error("Reinit failed for account %d, rolling back to %d: %s",
                  account_id, old_id, safe_exchange_error(e))
        await db.set_active_account(old_id)
        app_state.active_account_id = old_id
        exchange_factory.invalidate(account_id)
        # Reload old account's params/fees
        app_state.params = account_registry.get_account_params(old_id)
        try:
            await _reinit()  # re-init old account
        except Exception:
            log.critical("Rollback reinit also failed — entering degraded mode")
            app_state.is_initializing = False  # unblock UI, show dashboard-only
        raise
    # 4. Publish "risk:params_updated" event
```

**Files:** `api/routes_accounts.py` → `activate_account()`, `core/state.py` → `reset_for_account_switch()`
